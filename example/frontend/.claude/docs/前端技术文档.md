# 前端技术文档（Vue3+TS）
## 一、技术栈概述
本前端项目基于 Vue3 + TypeScript 开发，核心技术栈如下，所有技术使用需遵循本文档规范：
- 核心框架：Vue3（Composition API 风格，禁止使用Options API）
- 类型检查：TypeScript（严格模式，避免any类型滥用）
- 状态管理：Pinia（替代Vuex，简洁高效，支持持久化）
- 路由管理：Vue Router 4.x（实现页面跳转、权限控制）
- 接口请求：Axios（统一封装，拦截请求/响应）
- UI组件：Element Plus（统一UI风格，减少自定义样式）
- 样式框架：Tailwind CSS + SCSS（高效开发，样式统一）
- 构建工具：Vite（替代Webpack，提升构建速度）
- 代码规范：ESLint + Prettier（统一代码格式，避免语法错误）

## 二、核心技术使用规范
### 1. Vue3（Composition API）
- 组件编写：使用<script setup lang="ts">语法，简洁高效
- 响应式API：优先使用ref（基本类型）、reactive（引用类型），避免使用Vue2的data、methods
- 生命周期：使用onMounted、onUpdated等组合式API，替代Vue2的生命周期钩子
- 计算属性/监听：使用computed、watch，避免冗余逻辑，watch慎用（优先使用computed）
- 禁止事项：禁止使用Options API，禁止直接操作DOM（优先使用响应式API）

### 2. TypeScript
- 类型定义：所有变量、函数、组件props、接口返回值，均需定义明确类型，禁止滥用any
- 接口定义：使用interface定义对象类型（如接口请求/响应类型、组件props类型）
- 枚举使用：前端枚举与后端保持一致，放在src/enums目录，禁止重复定义
- 类型断言：谨慎使用as断言，避免类型欺骗，优先通过类型推导解决

### 3. Pinia状态管理
- 目录结构：按模块划分，每个模块一个store文件（如userStore.ts、settingStore.ts）
- 状态定义：使用defineStore定义store，state返回响应式数据，actions定义修改逻辑
- 持久化：全局状态（如用户信息）使用pinia-plugin-persistedstate持久化到localStorage
- 使用规范：
  - 全局状态：存放跨页面共用数据（用户信息、全局配置）
  - 局部状态：页面私有数据，使用组件内ref/reactive，不滥用全局状态
  - 禁止直接修改state，需通过actions修改，便于维护和调试

### 4. Vue Router
- 路由配置：放在src/router目录，按模块划分路由（如userRouter.ts、orderRouter.ts）
- 路由守卫：使用全局守卫实现权限控制（未登录跳转登录页），使用路由独享守卫控制单个路由权限
- 路由命名：遵循大驼峰命名（如UserList、Login），路径使用小写+横线（如/user/list）
- 懒加载：所有页面组件使用路由懒加载，减少首屏加载时间，示例：
  const UserList = () => import('../views/user/UserList.vue')

### 5. Axios接口请求
- 封装规范：放在src/api目录，按模块划分（如userApi.ts、orderApi.ts）
- 拦截器：统一配置请求拦截（添加token、请求头）、响应拦截（统一处理状态码、错误信息）
- 请求规范：
  - 接口命名：小驼峰，动词+名词（如getUserList、createOrder）
  - 参数传递：get请求用params，post/put请求用data
  - 异常处理：接口请求失败，统一使用Element Plus Message组件提示，避免重复代码

### 6. 样式规范（Tailwind CSS + SCSS）
- 全局样式：放在src/assets/styles目录，统一定义变量、公共样式
- 组件样式：使用scoped属性，避免样式污染，局部样式优先使用Tailwind工具类
- SCSS使用：用于定义全局变量、混合样式，避免重复编写样式
- 样式命名：遵循BEM命名规范（块-元素-修饰符），如.user-card__title--active

## 三、项目目录规范
src目录结构及用途，严格遵循以下组织方式，便于团队协作：
- src/api：接口封装目录，按业务模块划分
- src/components：可复用组件目录，按功能划分（如common、user、order）
- src/enums：枚举目录，与后端枚举保持一致
- src/assets：静态资源目录（styles：样式，images：图片，icons：图标）
- src/stores：Pinia状态管理目录，按模块划分
- src/router：路由配置目录，含路由守卫、路由模块
- src/views：页面组件目录，按业务模块划分（如user、order、home）
- src/utils：工具函数目录（request：Axios封装，errorHandle：异常处理，format：格式化工具）

## 四、注意事项
1.  所有代码需符合全局编码规范、命名规范，提交前执行npm run lint检查
2.  组件拆分遵循单一职责原则，粒度适中，可复用组件提取到src/components
3.  避免重复代码，可提取公共工具函数、公共组件，提升复用性
4.  接口请求、状态管理、路由跳转，均需遵循本文档规范，确保项目一致性
5.  与后端联调时，严格匹配接口参数、返回值类型，字段名与后端保持一致
