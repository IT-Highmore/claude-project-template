# 前端部署文档（Vue3+Vite）
## 一、部署环境准备
### 1. 本地开发环境（用于打包）
- 系统：Unix/macOS/Git Bash 或 Windows（PowerShell/CMD）
- 依赖：Node.js 16.x（必须，与package.json中依赖兼容）、npm 8.x+
- 工具：Git（拉取代码）、VS Code（可选，用于查看代码）

### 2. 服务器部署环境
- 系统：Linux（推荐CentOS 7+、Ubuntu 20.04+）
- 服务：Nginx（用于静态资源部署、反向代理）
- 端口：开放80端口（HTTP）、443端口（HTTPS，生产环境必备）
- 权限：服务器需有文件读写权限、端口开放权限

## 二、本地打包流程
### 1. 拉取代码（首次部署）
```bash
# 克隆前端Git仓库
git clone https://git.example.com/frontend.git
# 进入项目目录
cd frontend
# 切换到生产环境分支（main分支）
git checkout main
# 拉取最新代码
git pull
```

### 2. 安装依赖、打包项目
```bash
# 安装依赖（严格按照package.json中版本，避免版本冲突）
npm install
# 打包生产环境代码（生成dist目录）
npm run build
```

### 3. 打包检查
- 打包完成后，项目根目录会生成dist目录，包含所有静态资源（HTML、CSS、JS、图片）
- 检查dist目录是否完整，重点查看index.html、static目录是否存在
- 本地测试打包文件：使用live-server打开dist/index.html，确认页面正常显示、接口请求正常

## 三、服务器部署流程（Linux+Nginx）
### 1. 上传打包文件到服务器
- 方式1：使用scp命令上传（本地执行）
  ```bash
  scp -r ./dist 服务器用户名@服务器IP:/usr/share/nginx/html/frontend
  ```
- 方式2：使用FTP工具（如FileZilla），将dist目录上传到服务器指定路径（如/usr/share/nginx/html/frontend）
- 路径说明：/usr/share/nginx/html 为Nginx默认静态资源目录，frontend为项目目录

### 2. 配置Nginx（关键步骤）
#### （1）编辑Nginx配置文件
```bash
# 进入Nginx配置目录
cd /etc/nginx/conf.d
# 新建前端项目配置文件（frontend.conf）
vi frontend.conf
```

#### （2）配置文件内容（复制粘贴，修改对应参数）
```nginx
server {
    # 监听80端口（HTTP）
    listen 80;
    # 服务器域名（无域名则填服务器IP）
    server_name 你的域名/服务器IP;
    # 前端项目根目录（对应上传的dist目录路径）
    root /usr/share/nginx/html/frontend;
    # 默认首页
    index index.html;

    # 解决Vue Router单页应用刷新404问题
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存配置（提升访问速度）
    location ~* \.(js|css|png|jpg|jpeg|gif|ico)$ {
        expires 7d; # 缓存7天
        add_header Cache-Control "public, max-age=604800";
    }

    # 反向代理（解决跨域，转发后端接口请求）
    location /api {
        # 后端接口地址（替换为你的后端服务器IP:端口）
        proxy_pass http://后端IP:8080/api;
        # 保留请求头信息
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 日志配置（便于排查问题）
    access_log /var/log/nginx/frontend_access.log;
    error_log /var/log/nginx/frontend_error.log;
}
```

#### （3）检查Nginx配置、重启Nginx
```bash
# 检查配置文件是否有错误
nginx -t
# 重启Nginx，使配置生效
systemctl restart nginx
# 查看Nginx运行状态，确认启动成功
systemctl status nginx
```

### 3. 部署验证
- 打开浏览器，访问 服务器IP/域名，确认页面正常显示、接口请求正常
- 测试核心功能（登录、列表查询、表单提交），确认无异常
- 查看Nginx日志，排查是否有报错：tail -f /var/log/nginx/frontend_error.log

## 四、更新部署流程（后续版本更新）
1.  本地拉取最新代码：git pull
2.  重新打包：npm run build
3.  上传dist目录到服务器，覆盖原有frontend目录（建议先备份原有目录）
4.  重启Nginx：systemctl restart nginx
5.  验证更新内容，确认无异常

## 五、常见问题排查
1.  页面无法访问：检查Nginx是否启动、服务器端口是否开放、配置文件路径是否正确
2.  接口请求失败：检查Nginx反向代理配置（proxy_pass是否正确）、后端服务是否启动
3.  页面刷新404：检查Nginx配置中location / 的try_files配置是否正确
4.  静态资源加载失败：检查dist目录路径是否正确、静态资源文件是否完整
5.  样式错乱：检查打包时是否有报错、静态资源路径是否正确、浏览器缓存是否清除

## 六、注意事项
1.  生产环境建议配置HTTPS（申请SSL证书，修改Nginx配置监听443端口）
2.  部署前需确认后端服务已启动，接口可正常访问
3.  每次更新部署前，备份原有dist目录，避免部署失败无法回滚
4.  服务器需定期清理Nginx日志，避免日志文件过大
5.  本地打包时，确保npm install安装的依赖与package.json一致，避免打包报错
