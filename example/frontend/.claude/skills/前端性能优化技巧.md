# 前端性能优化技巧（Vue3+TS）
## 一、首屏加载优化（核心优化点）
### 1. 路由懒加载（必做）
- 原理：将页面组件拆分为单独的JS文件，只有访问该路由时才加载，减少首屏加载资源体积
- 实现方式（Vue Router）：
  ```typescript
  // src/router/index.ts
  const routes = [
    {
      path: '/login',
      name: 'Login',
      // 路由懒加载，自动拆分JS文件
      component: () => import('@/views/Login.vue')
    },
    {
      path: '/user/list',
      name: 'UserList',
      component: () => import('@/views/user/UserList.vue')
    }
  ]
  ```
- 优化效果：首屏JS体积减少60%以上，加载速度提升30%-50%

### 2. 静态资源优化
- 图片优化：
  - 压缩图片（使用TinyPNG等工具），减少图片体积
  - 使用WebP格式（比JPG/PNG小30%-50%），兼容低版本浏览器可降级为JPG/PNG
  - 图片懒加载：使用v-lazy指令（需安装vue3-lazyload），避免首屏加载所有图片
    ```vue
     ```
- 样式优化：
  - 优先使用Tailwind CSS，减少自定义CSS体积
  - 全局样式合并，避免多个CSS文件加载
  - 移除未使用的样式（使用PurgeCSS工具）

### 3. 依赖优化
- 移除冗余依赖：检查package.json，删除未使用的依赖（如废弃的插件）
- 替换重量级依赖：用轻量级依赖替代（如用dayjs替代moment.js）
- 按需引入依赖：Element Plus、Ant Design Vue等UI组件库，按需引入组件和样式，避免全量引入
  ```typescript
  // 按需引入Element Plus组件
  import { ElButton, ElInput } from 'element-plus'
  import 'element-plus/dist/index.css'
  ```

### 4. 打包优化（Vite）
- 配置vite.config.ts，压缩打包资源：
  ```typescript
  import { defineConfig } from 'vite'
  import vue from '@vitejs/plugin-vue'

  export default defineConfig({
    plugins: [vue()],
    build: {
      // 压缩JS/CSS
      minify: 'terser',
      // 拆分公共依赖，避免重复打包
      rollupOptions: {
        output: {
          manualChunks: {
            // 将Vue相关依赖拆分为单独的chunk
            vue: ['vue', 'vue-router', 'pinia'],
            // 将Element Plus拆分为单独的chunk
            elementPlus: ['element-plus']
          }
        }
      }
    }
  })
  ```

## 二、运行时性能优化
### 1. 减少组件重渲染（核心）
- 避免不必要的响应式数据：非响应式数据（如常量、固定配置），不使用ref/reactive包裹
- 使用computed缓存计算结果：复杂计算逻辑，使用computed缓存，避免每次渲染都重新计算
  ```typescript
  // 优化前：每次渲染都会执行计算
  const fullName = `${user.firstName} ${user.lastName}`
  // 优化后：只有user.firstName或user.lastName变化时，才重新计算
  const fullName = computed(() => `${user.firstName} ${user.lastName}`)
  ```
- 使用watch监听必要数据：避免watch监听整个对象，只监听需要变化的字段
  ```typescript
  // 优化前：监听整个user对象，任何字段变化都会触发
  watch(user, (newVal) => { ... })
  // 优化后：只监听user.name字段
  watch(() => user.name, (newVal) => { ... })
  ```
- 组件缓存：使用<KeepAlive>缓存频繁切换的组件（如标签页、弹窗），避免重复创建/销毁
  ```vue
  <KeepAlive>
    <router-view />
  </KeepAlive>
  ```

### 2. 事件优化
- 避免频繁触发的事件（如scroll、resize）：使用防抖（debounce）、节流（throttle）限制触发频率
  ```typescript
  import { debounce } from 'lodash-es'
  // 防抖：300ms内只触发一次
  const handleScroll = debounce(() => {
    // 滚动逻辑
  }, 300)
  window.addEventListener('scroll', handleScroll)
  ```
- 移除无用事件监听：组件卸载时，移除全局事件监听，避免内存泄漏
  ```typescript
  onUnmounted(() => {
    window.removeEventListener('scroll', handleScroll)
  })
  ```

### 3. 列表渲染优化
- 大数据列表（超过100条）：使用虚拟滚动（vue-virtual-scroller），只渲染可视区域的列表项
  ```vue
  {{ item.name }}</RecycleScroller>
  
  ```
- v-for优化：
  1.  必须绑定唯一key：使用数据的唯一标识（如id）作为key，避免使用index（index变化会导致所有列表项重渲染）
      ```vue
      <!-- 优化前：使用index作为key，不推荐 -->
      {{ item.name }}<!-- 优化后：使用唯一id作为key，推荐 -->
      {{ item.name }}
      ```
  2.  避免列表内嵌套复杂组件：将列表项拆分为独立的轻量级子组件，减少单个列表项的渲染成本
  3.  避免在v-for中使用v-if：若需筛选列表，优先在渲染前通过计算属性过滤数据，减少渲染量
      ```typescript
      // 优化前：v-for中嵌套v-if，每次渲染都筛选
      const list = ref([/* 原始数据 */])
      // 优化后：计算属性提前筛选，只渲染符合条件的项
      const filteredList = computed(() => list.value.filter(item => item.status === 1))
      ```
  4.  列表数据分页：大数据列表（超过500条），结合分页参数（pageNum、pageSize）实现分页加载，避免一次性渲染所有数据

### 4. 其他运行时优化
- 减少DOM操作：避免直接操作DOM（如document.querySelector、innerHTML），优先使用Vue响应式API
- 合理使用v-show和v-if：频繁切换显示/隐藏的元素用v-show（渲染一次，切换display），不频繁切换的用v-if（不渲染时移除DOM）
  ```vue
  <!-- 频繁切换（如弹窗）：v-show -->
  弹窗内容<!-- 不频繁切换（如权限控制的元素）：v-if -->
  有权限才显示的内容
  ```
- 优化计算逻辑：复杂逻辑（如数据格式化、筛选）提取到工具函数，避免在模板或组件内重复编写，提升复用性和维护性

## 三、代码层面优化
### 1. 代码拆分与复用
- 工具函数复用：将通用工具函数（如日期格式化、数据校验）提取到src/utils目录，避免重复编写
- 组件拆分：复杂页面拆分为多个子组件，遵循单一职责原则，提升代码可维护性和复用性
- 接口封装：统一封装Axios请求，提取公共请求拦截、响应拦截，避免每个接口重复编写拦截逻辑

### 2. 减少冗余代码
- 移除未使用的代码：定期清理未使用的变量、函数、组件，减少代码体积
- 避免重复逻辑：相同逻辑提取为公共函数或组件，如多个页面的表单校验逻辑、弹窗逻辑
- 简化条件判断：复杂条件判断使用对象映射替代if-else，提升代码可读性和执行效率
  ```typescript
  // 优化前：复杂if-else判断
  if (type === 1) {
    handleAdd()
  } else if (type === 2) {
    handleEdit()
  } else if (type === 3) {
    handleDelete()
  }
  // 优化后：对象映射，简洁高效
  const handleMap = {
    1: handleAdd,
    2: handleEdit,
    3: handleDelete
  }
  handleMap[type]?.()
  ```

## 四、缓存优化
### 1. 接口缓存
- 对于高频查询、数据不常变化的接口（如字典接口、配置接口），使用localStorage/sessionStorage缓存响应数据，避免重复请求
  ```typescript
  // 接口缓存示例
  const getDictList = async () => {
    // 先从localStorage获取缓存
    const cacheDict = localStorage.getItem('dictList')
    if (cacheDict) {
      return JSON.parse(cacheDict)
    }
    // 缓存不存在，请求接口
    const res = await dictApi.getDictList()
    // 缓存数据（设置过期时间，避免数据过时）
    localStorage.setItem('dictList', JSON.stringify(res.data))
    localStorage.setItem('dictExpire', Date.now() + 3600000) // 1小时过期
    return res.data
  }
  ```

### 2. 组件缓存
- 除了<KeepAlive>缓存路由组件，对于频繁复用的业务组件（如表格、表单），也可通过组件缓存减少重复渲染
- 避免过度缓存：不需要频繁切换、生命周期短的组件（如提示弹窗），无需缓存，避免占用内存

## 五、兼容性优化
### 1. 浏览器兼容性
- 适配主流浏览器（Chrome、Firefox、Edge、Safari），避免使用浏览器特有API
- 低版本浏览器（如IE11）适配：使用Babel转译ES6+语法，引入polyfill补充缺失的API
- 样式兼容性：使用Tailwind CSS的前缀自动生成功能，避免手动编写浏览器前缀

### 2. 设备适配
- 响应式适配：使用Tailwind的响应式类（sm:、md:、lg:、xl:），适配手机、平板、电脑等不同尺寸设备
- 移动端优化：避免使用过大的图片、复杂的动画，提升移动端加载速度和流畅度

## 六、优化效果验证
### 1. 常用验证工具
- Lighthouse：Chrome浏览器内置工具，可检测首屏加载速度、性能得分、优化建议
- Chrome开发者工具（Performance）：录制页面运行过程，分析重渲染、长任务、网络请求等瓶颈
- webpack-bundle-analyzer（Vite可用rollup-plugin-visualizer）：分析打包资源体积，定位冗余依赖

### 2. 优化指标参考
- 首屏加载时间：≤3秒（优秀），≤5秒（合格）
- 首次内容绘制（FCP）：≤1.8秒
- 最大内容绘制（LCP）：≤2.5秒
- 累积布局偏移（CLS）：≤0.1
- 长任务（超过50ms）：0个

## 七、注意事项
1.  性能优化需结合实际场景，避免过度优化（如简单页面无需复杂的打包拆分、缓存逻辑）
2.  优化过程中需兼顾代码可读性和可维护性，不能为了优化牺牲开发效率
3.  新增优化方案后，需测试验证优化效果，确保不影响原有功能
4.  定期复盘优化效果，根据项目迭代调整优化方案，持续提升页面性能
