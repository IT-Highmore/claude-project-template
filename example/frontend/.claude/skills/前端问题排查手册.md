## 一、常见报错排查（Vue3+TS）
### 1. 语法报错（编译阶段）
#### （1）TS类型错误（最常见）
- 报错示例：`Type 'string' is not assignable to type 'number'`
- 排查步骤：
  1.  定位报错位置，查看变量/参数的类型定义
  2.  确认赋值内容的类型与定义的类型是否一致
  3.  若需兼容多种类型，修改类型定义（如使用联合类型 `string | number`）
- 解决方案示例：
  ```typescript
  // 错误：类型不匹配
  const age: number = '20'
  // 正确：修改类型或赋值内容
  const age: number = 20
  // 或兼容多种类型
  const age: string | number = '20'
  ```

#### （2）组件导入/注册错误
- 报错示例：`Failed to resolve component: xxx`、`Component is not defined`
- 排查步骤：
  1.  检查组件导入路径是否正确（相对路径、绝对路径是否有误）
  2.  确认组件是否正确注册（全局注册/局部注册）
  3.  检查组件文件名与导入时的名称是否一致（区分大小写）
- 解决方案示例：
  ```typescript
  // 错误：路径错误、名称错误
  import UserForm from '@/components/Userform.vue' // 文件名是UserForm.vue，大小写错误
  // 正确
  import UserForm from '@/components/UserForm.vue'
  ```

#### （3）Vue指令使用错误
- 报错示例：`v-for requires a key attribute`、`Invalid v-model expression`
- 排查步骤：
  1.  v-for报错：确认v-for是否绑定了唯一key，key是否为字符串/数字
  2.  v-model报错：确认v-model绑定的是响应式数据（ref/reactive），自定义组件v-model是否正确配置
- 解决方案示例：
  ```vue
  <!-- 错误：v-for未绑定key -->
{{ item.name }}<!-- 正确 -->
  {{ item.name }}
  ```

### 2. 运行时报错（页面渲染/交互阶段）
#### （1）响应式数据相关报错
- 报错示例：`Cannot read properties of undefined (reading 'xxx')`
- 排查步骤：
  1.  确认报错的变量是否为响应式数据，是否已初始化
  2.  检查数据赋值时机（如接口请求未完成，就渲染数据）
  3.  避免在模板中直接访问深层未定义的属性（如 `user.address.city`，需先判断user.address是否存在）
- 解决方案示例：
  ```vue
  <!-- 错误：直接访问深层未定义属性 -->
  {{ user.address.city }}<!-- 正确：使用可选链操作符 -->
  {{ user?.address?.city || '无' }}
  ```

#### （2）事件绑定错误
- 报错示例：`handleClick is not a function`
- 排查步骤：
  1.  检查事件绑定的方法名是否与定义的方法名一致（区分大小写）
  2.  确认方法是否在`
