# 安全规范（前后端共用）
1. 前端安全：
   - 接口请求需携带token，放在请求头Authorization中，使用Axios拦截器统一添加。
   - 输入框内容校验，过滤特殊字符，防止XSS攻击，使用Element Plus的表单校验功能。
   - 不暴露敏感信息（如密钥、token）在前端代码中，敏感配置放在.env文件中。
   - 避免直接操作DOM，使用Vue3响应式API。

2. 后端安全：
   - 接口权限校验，使用Spring Security，不同角色分配不同访问权限，基于RBAC模型。
   - 数据库操作使用MyBatis-Plus参数化查询，防止SQL注入，禁止拼接SQL。
   - 敏感字段（如密码）使用BCrypt加密存储，不明文保存，密码长度不小于8位。
   - 接口请求频率限制，使用Redis实现限流，防止恶意请求，单个IP每分钟不超过60次请求。
   - 全局异常拦截，统一处理异常，不暴露具体错误信息给前端。

3. 通用安全：
   - 避免使用过期依赖，及时更新依赖包，修复安全漏洞，前端定期更新npm依赖，后端定期更新Maven依赖。
   - 日志记录关键操作（如登录、权限变更、接口异常），使用SLF4J+Logback记录日志，便于排查问题。
   - 前后端数据传输使用HTTPS，确保数据安全，开发环境可使用HTTP，生产环境必须使用HTTPS。
