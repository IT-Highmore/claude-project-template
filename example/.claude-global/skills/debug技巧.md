# 通用调试技巧
## 一、前后端联调调试技巧
### 1. 接口请求调试
- 前端：使用浏览器开发者工具（F12）→ Network，查看请求参数、请求头、响应数据，排查参数传递异常
  - 重点关注：请求方式、Content-Type、token是否正确，参数名称、类型是否与后端一致
  - 异常处理：若响应400，检查参数格式；401检查token；500查看后端日志
- 后端：使用Postman/Swagger自测接口，模拟前端请求，排查接口逻辑、参数校验异常
  - 重点关注：参数接收是否正确、业务逻辑是否符合需求、数据库操作是否正常

### 2. 数据不一致调试
- 现象：前端展示数据与后端返回数据不一致、后端返回数据与数据库数据不一致
- 排查步骤：
  1.  前端打印请求响应数据，确认后端返回是否正确
  2.  后端打印接口入参、出参，确认业务逻辑处理是否正确
  3.  检查数据库查询语句，确认数据查询是否正确
  4.  检查前后端枚举、字段映射是否一致

### 3. 跨域问题调试
- 现象：前端请求接口提示CORS错误，无法获取响应
- 排查步骤：
  1.  确认后端是否配置跨域（Spring Boot配置CorsConfiguration）
  2.  检查前端请求头是否包含自定义字段，后端是否允许该字段
  3.  确认跨域配置是否覆盖所有接口前缀（/api/v1）

## 二、Claude使用调试技巧
1.  规则加载调试：输入 /memory 指令，查看Claude是否正确加载全局规则和项目独有规则
2.  联动同步调试：修改前端/后端代码后，提示Claude同步，查看是否正确修改对应代码
  - 若同步失败，检查符号链接是否正常、项目路径是否正确
3.  代码生成调试：让Claude生成代码后，先检查是否符合编码规范、是否适配技术栈，再复制使用
  - 若生成代码有语法错误，提示Claude修正，并补充技术栈约束

## 三、通用调试工具推荐
1.  前端调试：浏览器开发者工具（Network/Console/Elements）、Vue DevTools（Vue3调试）
2.  后端调试：IDEA调试模式（断点、变量监控）、Postman（接口自测）
3.  数据库调试：Navicat/DBeaver（查询、修改数据）、MySQL日志（排查SQL异常）
4.  日志调试：后端SLF4J+Logback日志、前端console.log，重点记录关键操作和异常信息

## 四、常见问题调试总结
1.  接口请求无响应：检查后端服务是否启动、接口路径是否正确、网络是否正常
2.  代码报错：先查看报错信息，定位报错位置，再排查语法、逻辑、依赖问题
3.  数据查询异常：检查SQL语句、参数传递、数据库连接是否正常
4.  Claude无法访问项目代码：检查符号链接是否有效、项目路径是否与配置一致
