# 依赖版本说明
## 一、版本规范说明
1.  版本格式：主版本号.次版本号.修订号（如1.2.3）
    - 主版本号：重大功能变更，不兼容旧版本
    - 次版本号：新增功能，兼容旧版本
    - 修订号：bug修复，兼容旧版本
2.  依赖选择原则：优先选择稳定版本（非beta、alpha版本），避免使用最新未验证版本
3.  版本统一：前后端各依赖版本统一，避免版本冲突，新增依赖需确认团队成员一致使用

- 版本冲突解决：使用npm ls 依赖名 查看冲突版本，指定兼容版本

## 四、注意事项
1.  前端npm依赖、后端Maven依赖，均需在对应配置文件（package.json、pom.xml）中明确标注版本
2.  新增依赖时，需测试兼容性，确保不影响现有功能
3.  禁止使用过期依赖（存在安全漏洞、不再维护的版本）
4.  团队成员同步依赖版本，避免本地依赖与项目配置不一致
