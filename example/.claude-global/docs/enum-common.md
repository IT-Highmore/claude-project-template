# 通用枚举文档（前后端共用）
## 说明
本文档定义前后端共用枚举，所有枚举值需保持完全一致（名称、值、说明），修改时双向同步，前端放在src/enums，后端放在com.example.backend.enums。

## 一、基础通用枚举
### 1. 通用状态枚举（CommonStatusEnum）
| 枚举值 | 枚举名称 | 说明 | 适用场景 |
|--------|----------|------|----------|
| 0      | DISABLE  | 禁用 | 账号、菜单、接口等状态控制 |
| 1      | ENABLE   | 启用 | 账号、菜单、接口等状态控制 |

### 2. 性别枚举（GenderEnum）
| 枚举值 | 枚举名称 | 说明 | 适用场景 |
|--------|----------|------|----------|
| 0      | UNKNOWN  | 未知 | 用户性别 |
| 1      | MALE     | 男   | 用户性别 |
| 2      | FEMALE   | 女   | 用户性别 |

### 3. 数据删除枚举（DeleteEnum）
| 枚举值 | 枚举名称 | 说明 | 适用场景 |
|--------|----------|------|----------|
| 0      | NOT_DELETE | 未删除 | 所有表的逻辑删除标识 |
| 1      | DELETE    | 已删除 | 所有表的逻辑删除标识 |

## 二、业务通用枚举
### 1. 订单状态枚举（OrderStatusEnum）
| 枚举值 | 枚举名称 | 说明 |
|--------|----------|------|
| 10     | PENDING  | 待支付 |
| 20     | PAID     | 已支付 |
| 30     | SHIPPED  | 已发货 |
| 40     | COMPLETED | 已完成 |
| 50     | CANCELLED | 已取消 |

### 2. 接口请求类型枚举（RequestTypeEnum）
| 枚举值 | 枚举名称 | 说明 | 对应请求方式 |
|--------|----------|------|--------------|
| 1      | QUERY    | 查询 | GET |
| 2      | CREATE   | 新增 | POST |
| 3      | UPDATE   | 修改 | PUT |
| 4      | DELETE   | 删除 | DELETE |

## 三、枚举使用规范
1.  枚举命名：大驼峰+Enum后缀（如 CommonStatusEnum），符合全局命名规范
2.  枚举值：优先使用数字类型，避免字符串，便于数据库存储和传输
3.  新增共用枚举：需在本文档添加，通知前后端开发人员同步更新
4.  修改枚举：需同步修改本文档、前端枚举、后端枚举，确保一致性
