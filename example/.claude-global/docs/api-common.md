# 通用接口文档（前后端共用）
## 一、接口基础规范
1.  接口前缀：所有接口统一前缀 /api/v1（版本迭代时新增 /api/v2，保持向下兼容）
2.  请求方式：
    - GET：查询类接口（无副作用，可缓存）
    - POST：新增类接口（有副作用，不可缓存）
    - PUT：修改类接口（全量更新，有副作用）
    - DELETE：删除类接口（有副作用）
3.  数据格式：请求/响应均为 JSON 格式，编码 UTF-8
4.  请求头要求：
    - 认证token：Authorization: Bearer {token}（登录后所有接口必传）
    - 内容类型：Content-Type: application/json

## 二、统一响应格式
### 1. 成功响应
{
  "code": 200,
  "message": "success",
  "data": {}  // 具体返回数据，根据接口需求定义
}

### 2. 失败响应
{
  "code": 状态码,
  "message": "错误描述（简洁明了，便于前端提示）",
  "data": null  // 失败时默认null
}

## 三、统一状态码定义
| 状态码 | 含义 | 适用场景 |
|--------|------|----------|
| 200    | 成功 | 所有正常响应 |
| 400    | 参数错误 | 请求参数缺失、格式错误 |
| 401    | 未授权 | 未登录、token过期、token无效 |
| 403    | 无权限 | 登录用户无该接口访问权限 |
| 404    | 资源不存在 | 访问的接口、数据不存在 |
| 500    | 服务器错误 | 后端代码异常、数据库异常 |
| 503    | 服务不可用 | 后端服务未启动、部署异常 |

## 四、通用参数说明
1.  分页参数（所有列表查询接口统一）：
    - pageNum：页码（默认1）
    - pageSize：每页条数（默认10，最大50）
2.  排序参数：
    - sortField：排序字段（需与数据库字段一致）
    - sortOrder：排序方式（asc=升序，desc=降序）
3.  模糊查询参数：
    - keyword：通用模糊查询关键词（适配名称、编号等字段）

## 五、接口联调注意事项
1.  后端接口开发完成后，需更新Swagger文档，标注参数、返回值、备注
2.  前端请求接口时，需严格匹配请求方式、参数格式，统一处理响应状态码
3.  接口字段变更时，前后端需同步修改，后端需提前通知前端
