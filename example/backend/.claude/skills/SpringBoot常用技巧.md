# Spring Boot 常用技巧

## 一、接口层

- 控制层只接收和返回，不写复杂业务
- 参数对象优先使用 DTO，避免直接暴露 Entity
- 批量接口优先明确限制条数，避免一次性提交过大请求

## 二、校验

- 使用 `@NotNull`、`@NotBlank`、`@Size` 等注解做基础校验
- 复杂规则放到 Service，不要把业务校验全部塞到注解里
- 校验错误统一返回标准错误结构

## 三、事务

- 同一业务动作跨多表时再加 `@Transactional`
- 对外部 HTTP 调用、消息发送等副作用操作要考虑事务边界
- 事务方法不要滥用 `private`，避免代理失效

## 四、查询与分页

- 列表接口统一分页参数
- 高频列表接口优先补索引，再考虑缓存
- 避免 Controller 中拼装查询条件

## 五、缓存

- 只缓存高频、低实时性或代价高的数据
- Key 统一格式：`system:module:biz:id`
- 更新数据后同步考虑缓存失效策略

## 六、文档

- 每次接口变更同步更新 Swagger 注解
- 对前端容易误解的字段补充备注
- 对枚举字段写清楚可选值和含义
