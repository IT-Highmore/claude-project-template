# 后端问题排查手册

## 一、应用启动失败

优先检查：

1. JDK 版本是否匹配
2. `application-*.yml` 是否缺少必填配置
3. 端口是否被占用
4. 数据库、Redis 是否可连接

## 二、接口 401 / 403

优先检查：

1. Token 是否携带
2. Token 是否过期
3. 网关或安全配置是否拦截
4. 角色权限是否缺失

## 三、数据库连接异常

优先检查：

1. 数据库地址、端口、用户名、密码
2. 连接池配置是否过小
3. 数据库白名单和网络策略
4. SQL 执行是否超时

## 四、接口变慢

优先检查：

1. 慢 SQL
2. 缺失索引
3. 大分页或全表扫描
4. 外部接口调用超时
5. 缓存未命中或缓存雪崩

## 五、联调异常

优先检查：

1. 前端请求参数是否与 DTO 一致
2. 返回字段是否与前端类型一致
3. 跨域配置是否正确
4. Swagger 文档是否已更新

## 六、上线后故障

优先处理顺序：

1. 判断是否能快速回滚
2. 保留错误日志和现场信息
3. 缩小影响范围
4. 修复后补充复盘和文档
