# 数据库设计检查清单

## 一、命名

- 表名使用小写下划线
- 主键统一命名风格，如 `id`
- 外键字段统一使用 `xxx_id`

## 二、字段设计

- 字段名要表达业务含义
- 金额、数量、状态等字段类型要明确
- 时间字段统一考虑时区和精度
- 是否删除、是否启用等状态字段要明确默认值

## 三、索引

- 列表筛选字段是否建立索引
- 联表字段是否建立索引
- 唯一业务约束是否建立唯一索引
- 避免无效索引和重复索引

## 四、一致性

- DTO、VO、Entity 字段是否一致
- 前端表单字段和数据库字段映射是否明确
- 枚举值和数据库存储值是否一一对应

## 五、变更发布

- 是否准备迁移脚本
- 是否评估历史数据兼容
- 是否评估回滚方案
- 是否通知前端字段变更影响
