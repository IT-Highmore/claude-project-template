# 后端技术文档（Spring Boot + Java）

## 一、技术栈约定

- JDK：11
- 框架：Spring Boot 2.7.x
- Web：Spring MVC
- 持久层：MyBatis-Plus
- 数据库：MySQL 8.0
- 缓存：Redis 6.x
- 文档：Swagger / Knife4j
- 构建：Maven

## 二、推荐包结构

```text
src/main/java/com/example/backend/
├─ controller/
├─ service/
├─ service/impl/
├─ repository/
├─ dto/
├─ vo/
├─ entity/
├─ enums/
├─ config/
├─ exception/
└─ common/
```

## 三、职责划分

### Controller

- 只做参数接收、权限控制、结果返回
- 不直接编排复杂业务
- 不直接操作数据库

### Service

- 处理业务规则和事务边界
- 负责跨表逻辑、状态流转和领域校验
- 向上返回 DTO / VO，不泄露底层实现细节

### Repository

- 负责 CRUD 和查询封装
- 复杂查询集中到 Mapper XML 或专用查询对象
- 不承载业务判断

## 四、统一实践

### 参数校验

- 入参统一使用 JSR-380 注解校验
- 控制层开启 `@Valid` 或 `@Validated`
- 校验失败统一进入全局异常处理

### 返回结构

- 成功和失败统一返回标准结构
- 不直接把异常堆栈暴露给前端
- 错误码与错误文案要可追踪、可定位

### 日志

- 关键链路记录 `INFO`
- 异常和降级记录 `WARN` / `ERROR`
- 日志中避免输出密码、Token、身份证号等敏感信息

### 事务

- 只有真正跨资源或跨表的一致性场景才加事务
- 查询方法默认不加事务
- 避免在大循环、外部调用、异步逻辑中滥用事务

## 五、与前端联动时的约定

- 修改字段名时，同步确认 DTO、VO、前端类型、表单和枚举
- 修改分页字段时，同步确认前端列表页、筛选条件和排序参数
- 修改错误码时，同步确认前端错误提示和重试逻辑
- 修改返回结构时，同步更新 Swagger 文档和联调说明
