# 后端部署文档

## 一、环境划分

- `dev`：本地开发与联调
- `test`：测试环境
- `prod`：生产环境

建议每个环境都使用独立的数据库、缓存和配置文件。

## 二、基础要求

- 安装 JDK 11
- 安装 Maven 3.8+
- 准备 MySQL 8.0 和 Redis 6.x
- 确认应用所需端口、日志目录、上传目录已创建

## 三、本地启动

```bash
mvn clean package -DskipTests
java -jar target/backend.jar --spring.profiles.active=dev
```

## 四、配置管理建议

- 基础配置放 `application.yml`
- 环境差异放 `application-dev.yml`、`application-test.yml`、`application-prod.yml`
- 密钥、密码、第三方凭证通过环境变量或密钥系统注入

## 五、上线前检查

1. 数据库脚本是否执行
2. Swagger 是否与代码一致
3. 日志路径和文件权限是否正常
4. 跨域、回调地址、文件存储路径是否正确
5. 前端依赖的接口是否完成回归

## 六、回滚建议

- 保留最近两个稳定包
- 数据库变更优先设计可回滚方案
- 发布记录保留版本号、发布时间、负责人和变更摘要

## 七、联调提醒

- 上线前确认前端使用的接口版本
- 返回字段变更要先通知前端
- 鉴权、跨域、上传下载等高风险链路必须单独验证
